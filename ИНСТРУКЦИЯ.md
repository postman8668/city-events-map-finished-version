# Инструкция по использованию системы модерации событий

## Для пользователей

### Как создать событие:

1. **Войдите в систему** под своим аккаунтом
2. В верхнем меню нажмите **"Мои события"**
3. Нажмите кнопку **"Создать событие"**
4. Заполните форму:
   - Название события
   - Описание
   - Дата и время
   - Место проведения
   - Координаты на карте
   - Категория
   - Интересы (теги)
   - Цена (0 для бесплатных)
   - Максимальное количество участников
5. Нажмите **"Создать"**

### Статусы событий:

- 🟡 **На модерации** - событие отправлено администратору на проверку
- 🟢 **Одобрено** - событие опубликовано и видно всем пользователям
- 🔴 **Отклонено** - событие не прошло модерацию

### Просмотр своих событий:

- Перейдите в **"Мои события"** в верхнем меню
- Здесь вы увидите все созданные вами события с их статусами
- Вы можете редактировать свои события, нажав кнопку **"Редактировать"**

---

## Для администратора

### Модерация событий:

1. **Войдите как администратор** (username: admin)
2. Перейдите в **"Админ" → "События"**
3. Используйте фильтры:
   - **Все события** - показать все
   - **На модерации** - события, ожидающие проверки (с красным значком)
   - **Одобренные** - опубликованные события
   - **Отклоненные** - отклоненные события

### Действия с событиями на модерации:

- ✅ **Одобрить** - событие будет опубликовано и станет видно всем
- ❌ **Отклонить** - событие будет отклонено
- ✏️ **Редактировать** - изменить данные события
- 🗑️ **Удалить** - удалить событие полностью

### Создание событий администратором:

- События, созданные администратором, публикуются **сразу без модерации**
- Используйте кнопку **"Создать событие"** в разделе "События"

### Панель управления:

- В админ-панели отображается:
  - Общее количество событий
  - Количество событий на модерации (с предупреждением)
  - Количество пользователей
  - Количество записей в логах

---

## Важные замечания:

1. **Обычные пользователи** не могут редактировать чужие события
2. **Администратор** может редактировать и удалять любые события
3. Все действия записываются в **логи системы**
4. На главной странице показываются **только одобренные события**
5. События на модерации и отклоненные **не видны обычным пользователям**

---

## Миграция существующей базы данных:

Если вы обновляете систему и у вас уже есть события в базе данных:

```bash
python migrate_db.py
```

Этот скрипт:
- Добавит необходимые поля в базу данных
- Установит всем существующим событиям статус "Одобрено"
- Не удалит никакие данные
